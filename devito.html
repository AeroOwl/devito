devito.compiler.get_compiler_from_env()[source]¶

Derive compiler class and settings from environment variables

Returns:The compiler indicated by the environment variable.
The key environment variable DEVITO_ARCH supports the following values:
  • ‘gcc’ or ‘gnu’ - (Default) Standard GNU compiler toolchain
  • ‘clang’ or ‘osx’ - Clang compiler toolchain for Mac OSX
  • ‘intel’ or ‘icpc’ - Intel compiler toolchain via icpc
  • ‘intel-mic’ or ‘mic’ - Intel MIC using offload mode via pymic

Additionally, the variable DEVITO_OPENMP can be used to enable OpenMP parallelisation on by setting it to “1”.

devito.finite_difference.second_derivative(*args, **kwargs)[source]¶

Derives second derivative for a product of given functions.

Parameters:
  • *args – All positional arguments must be fully qualified function objects, eg. f(x, y) or g(t, x, y, z).
  • dim – Symbol defininf the dimension wrt. which to differentiate, eg. x, y, z or t.
  • diff – Finite Difference symbol to insert, default h.
  • order – Discretisation order of the stencil to create.
Returns:

The second derivative
Example: Deriving the second derivative of f(x, y)*g(x, y) wrt. x via
second_derivative(f(x, y), g(x, y), order=2, dim=x) results in (-2.0*f(x, y)*g(x, y) + 1.0*f(-h + x, y)*g(-h + x, y) + 1.0*f(h + x, y)*g(h + x, y)) / h**2.
devito.finite_difference.cross_derivative(*args, **kwargs)[source]¶

Derives corss derivative for a product of given functions.

Parameters:
  • *args – All positional arguments must be fully qualified function objects, eg. f(x, y) or g(t, x, y, z).
  • dims – 2-tuple of symbols defining the dimension wrt. which to differentiate, eg. x, y, z or t.
  • diff – Finite Difference symbol to insert, default h.
Returns:

The cross derivative
Example: Deriving the cross-derivative of f(x, y)*g(x, y) wrt. x and y via:
cross_derivative(f(x, y), g(x, y), dims=(x, y)) results in: 0.5*(-2.0*f(x, y)*g(x, y) + f(x, -h + y)*g(x, -h + y) + f(x, h + y)*g(x, h + y) + f(-h + x, y)*g(-h + x, y) - f(-h + x, h + y)*g(-h + x, h + y) + f(h + x, y)*g(h + x, y) - f(h + x, -h + y)*g(h + x, -h + y)) / h**2

class devito.operator.Operator(nt, shape, dtype=<type 'numpy.float32'>, stencils=[], substitutions=[], spc_border=0, time_order=0, forward=True, compiler=None, profile=False, cache_blocking=False, block_size=5, input_params=None, output_params=None, factorized={})[source]¶

Bases: object

Class encapsulating a defined operator as defined by the given stencil

The Operator class is the core abstraction in DeVito that allows users to generate high-performance Finite Difference kernels from a stencil definition defined from SymPy equations.

Parameters:
  • nt – Number of timesteps to execute
  • shape – Shape of the data buffer over which to execute
  • dtype – Data type for the grid buffer
  • stencils – SymPy equation or list of equations that define the stencil used to create the kernel of this Operator.
  • substitutions – Dict or list of dicts containing the SymPy symbol substitutions for each stencil respectively.
  • spc_border – Number of spatial padding layers
  • time_order – Order of the time discretisation
  • forward – Flag indicating whether to execute forward in time
  • compiler – Compiler class used to perform JIT compilation. If not provided, the compiler will be inferred from the environment variable DEVITO_ARCH, or default to GNUCompiler.
  • profile – Flag to enable performance profiling
  • cache_blocking – Flag to enable cache blocking
  • block_size – Block size used for cache clocking. Can be either a single number used for all dimensions or a list stating block sizes for each dimension. Set block size to None to skip blocking on that dim
  • input_params – List of symbols that are expected as input.
  • output_params – List of symbols that define operator output.
  • factorized – A map given by {string_name:sympy_object} for including factorized terms

Change all terms from function notation to array notation.

Finds all terms of the form term(x1, x2, x3) and changes them to repl[x1, x2, x3]. i.e. changes from function notation to array notation. It also reorders the indices x1, x2, x3 so that the time index comes first.
