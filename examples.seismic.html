examples.seismic.model module¶

class examples.seismic.model.Model(origin, spacing, shape, vp, nbpml=0, dtype=<type 'numpy.float32'>, epsilon=None, delta=None, theta=None, phi=None)[source]¶

Bases: object

The physical model used in seismic inversion processes.

Parameters:
  • origin – Origin of the model in m as a tuple in (x,y,z) order
  • spacing – Grid size in m as a Tuple in (x,y,z) order
  • shape – Number of grid points size in (x,y,z) order
  • vp – Velocity in km/s
  • nbpml – The number of PML layers for boundary damping
  • rho – Density in kg/cm^3 (rho=1 for water)
  • epsilon – Thomsen epsilon parameter (0<epsilon<1)
  • delta – Thomsen delta parameter (0<delta<1), delta<epsilon
  • theta – Tilt angle in radian
  • phi – Asymuth angle in radian

The Model provides two symbolic data objects for the creation of seismic wave propagation operators:

Parameters:
  • m – The square slowness of the wave
  • damp – The damping field for absorbing boundarycondition
critical_dt¶

Critical computational time step value from the CFL condition.

dim¶

Spatial dimension of the model domain

domain_size¶

Physical size of the domain as determined by shape and spacing

get_spacing()[source]¶

Return the grid size

pad(data)[source]¶

Padding function PNL layers in every direction for for the absorbing boundary conditions.

:param data : Data array to be padded

shape_domain¶

Computational shape of the model domain, with PML layers

vp¶

numpy.ndarray holding the model velocity in km/s.

Updating the velocity field also updates the square slowness self.m. However, only self.m should be used in seismic operators, since it is of type DenseData.

examples.seismic.model.demo_model(preset, **kwargs)[source]¶

Utility function to create preset Model objects for demonstration and testing purposes. The particular presets are

* 'layer2D': Simple two-layer model with velocities 1.5 km/s
and 2.5 km/s in the top and bottom layer respectively.
  • ‘marmousi2D’: Loads the 2D Marmousi data set from the given
    filepath. Requires the opesci/data repository to be available on your machine.

class examples.seismic.source.PointSource(*args, **kwargs)[source]¶

Bases: devito.pointdata.PointData

Symbolic data object for a set of sparse points sources

Parameters:
  • name – Name of the symbol representing this source
  • coordinates – Point coordinates for this source
  • data – (Optional) Data values to initialise point data
  • ntime – (Optional) Number of timesteps for which to allocate data
  • npoint – (Optional) Number of sparse points represented by this source
  • dimension – :(Optional) class:Dimension object for representing the number of points in this source

Note, either the dimensions ntime and npoint or the fully initialised data array need to be provided.
